Login Kit & User Data

  • Login kit provides avatar and user data, not authentication functionality ‣
  • Enabling login kit provides union ID in addition to existing data ‣
  • TikTok provides two types of IDs: open ID and union ID ‣
  • Device ID may be useful for identifying account ban patterns — if device ID is the same across banned and new accounts, this could explain why bans occur ‣
  • Follower information is now accessible but has limited utility ‣ ‣

TikTok API Research & Challenges

  • Applied for 10 different APIs, with 1 rejection ‣
  • App category selection is critical — wrong category prevents access to specific APIs, and categories cannot be added after app creation ‣
  • Current app is in draft status and limited to sandbox mode with specific test accounts ‣ ‣
  • To move beyond sandbox, app must go through submission and review process ‣ ‣
  • APIs are filtered during review — team will only receive access to APIs that make sense for the app's use case ‣

API Types & Data Access

  • Discovery of partner API v2 and connector app type ‣ ‣
  • Connector apps are designed for integrations like Salesforce to TikTok ‣
  • Partner center API may not be available in US region — some features appear UK/EU only ‣
  • Likely approach: combining official API with scraping and seller center exports ‣

Desired Data & Use Cases

  • Need access to comprehensive seller/partner center data including: ‣ ‣ ‣
    • Video performance metrics (views, completion rate)
    • Sales data per video (units sold, revenue)
    • Direct links to videos
    • Correlation between posting frequency and sales
  • Example: video with 12,000 views and ~50% completion rate generated 27 sales ‣ ‣
  • This data would enable video breakdown analysis to identify successful content patterns ‣

Potential Solutions
