Data Access Challenges and Solutions

  • Primary goal is to access TikTok Shop data (sales metrics) which is not available through standard APIs ‣ ‣
  • Partner Center provides access to creator data but only through web interface with export functionality, no direct API ‣ ‣
  • Proposed bookmarklet solution: create a browser bookmark that scrapes visible data from Partner Center pages when clicked ‣ ‣ ‣
  • Bookmarklet would be semi-automated, requiring someone to click daily but still viable ‣ ‣
  • Alternative approach: use Elasticsearch/GreyLog to aggregate data from disparate sources as single source of truth ‣ ‣

Competitor Analysis - Yuca.ai

  • Yuca.ai is the primary competitor to mimic, offering campaign hosting and video metrics tracking ‣
  • Their platform shows creator login (using TikTok login kit), campaign management for brands, and video performance metrics ‣
  • Console inspection revealed extensive third-party integrations with many errors, particularly from PostHog analytics ‣ ‣ ‣
  • Plan to inspect Yuca.ai's network calls to identify which APIs they're using ‣

Application Development Updates

  • Currently working on second version of the website with improved security ‣ ‣
  • Built on Superstarter foundation using BetterAuth and Supabase for secure authentication and API connections ‣ ‣
  • Implemented rule to never deviate from foundation, only extend it ‣ ‣
  • Dashboard includes connected accounts management, goals/campaign tracking, and video playback functionality ‣ ‣ ‣
  • Initial page load speed is the main performance concern, though in-session navigation is fast ‣ ‣
  • Using AI-powered development tools (Cursor) to build 80-90% of features, with improved hallucination handling ‣ ‣

Campaign Management Strategy

  • Two primary models: Yuca-style campaigns (tracked through app) and Discord campaigns (submission-based) ‣ ‣
  • Discord is where major brands currently communicate with TikTok creators ‣ ‣
  • Goal is to aggregate all campaign information in their app so creators don't need to leave platform ‣ ‣
  • Need to verify posted video counts against actual data for accuracy ‣

MVP Definition and Scope

  • MVP focused on login functionality and displaying all user data ‣ ‣
  • Public API data (likes, views, comments) are nice-to-have but not immediately beneficial ‣ ‣
  • Core value is TikTok Shop data showing sales correlated with video performance ‣
  • If Partner Center OAuth works, implementation would involve sending links to creators to connect accounts ‣

Platform Expansion Considerations

  • Currently TikTok-focused but planning to expand to other social platforms adopting similar creator economy features ‣
  • Discussed affiliate marketing progression from basic link sharing to full funnel ownership ‣ ‣
  • Mobile optimization prioritized over native app due to 45+ year old target demographic and TikTok's mobile-first nature ‣
  • PWA (Progressive Web App) considered sufficient for mobile experience instead of iOS/Android native apps ‣ ‣
