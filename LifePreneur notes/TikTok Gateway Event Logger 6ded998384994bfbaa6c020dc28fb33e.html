Strategic Direction

  • Pivoting away from desktop solutions: Team decided to abandon bookmarklet and browser extension approaches in favor of focusing solely on mobile app development ‣ ‣
  • App Store as primary distribution: The goal is to have approved apps in both Apple App Store and Google Play Store, creating trust and discoverability ‣ ‣
  • Accepting Apple's fees: Team is willing to let Apple take 25% commission on memberships sold through the App Store, treating Apple as an affiliate ‣ ‣
  • Comprehensive app vision: The app will combine analytics data from TikTok accounts with membership content including training videos, creating an all-in-one experience ‣ ‣ ‣

Three-Phase Rollout Plan

Phase 1 - Internal Testing: Launch with team members to test functionality and connect accounts ‣

Phase 2 - Private Beta: Roll out to small group of active existing members for testing ‣ ‣

Phase 3 - Public App Store Launch: Submit to App Store for public availability once stable ‣ ‣

  • New members on free trial won't need early access since they lack historical analytics to review ‣
  • Using DenoDeploy to maintain separate environments (QA, dev, beta, production) with unique URLs and databases for each phase ‣ ‣

Technical Implementation

App Architecture:

  • Starting with vanilla JavaScript and Svelte for rendering ‣
  • Using Cordova to wrap web content for Android and iOS ‣
  • Can also build for desktop (Mac, Windows) and PWA with service workers ‣

Account Connection Flow:

  • Users connect multiple TikTok accounts through the app ‣ ‣
  • Each account shows verification status ‣
  • Accounts page displays all connected accounts with option to disconnect ‣
  • Refresh functionality for when login tokens expire ‣
  • Each connected account can serve as a login method ‣ ‣

Analytics Display:

  • Default view shows combined stats across all connected accounts ‣
  • Users can then drill down into individual account data ‣
  • Focus on meaningful analytics at launch, with ability to add more over time ‣

Authentication Approach:

  • Currently blocked on TikTok login implementation ‣
  • Considering creating standalone LifeNor account system instead of OAuth ‣
  • Would allow expansion to other social platforms (Facebook, Instagram) ‣
  • Each connected social account could also serve as login credential ‣

App Store Considerations

Timeline Expectations:

  • Android approval is faster, Apple takes minimum 15 days with perfect submission ‣ ‣
  • 30-day timeline before public launch is acceptable ‣
  • Team has 15+ day buffer by using internal and beta testing phases first ‣

Submission Requirements:

  • All icons, legal policies, and details must be perfect for Apple ‣
  • Process described as "soul-sucking" but necessary ‣ ‣
  • Apple requires separate developer account for each app ‣
  • Google allows multiple apps under one account ‣

App Store Benefits:

  • Creates trust through official distribution channel ‣
  • Provides discoverability through App Store search ‣
  • Allows user reviews and testimonials (though with less control) ‣

Payment Integration:

  • TikTok example shows they encourage web purchases to avoid Apple's 25% fee ‣ ‣
  • Team is comfortable with Apple taking commission on App Store signups since Apple becomes the acquisition channel ‣ ‣
  • Existing members joining through other channels wouldn't trigger Apple fees ‣
  • Affiliate links would be processed before App Store to maintain affiliate credit ‣ ‣

connectWebSocket() {
        const websocketUrl = 'ws://localhost:21213';
        console.log(`Attempting to connect to WebSocket: ${websocketUrl}`);
        this.ws = new WebSocket(websocketUrl);

        this.ws.onopen = () => {
            console.log('Connected to TikFinity WebSocket!');
            this.addEventToLog('System: Connected to TikFinity!', '#00ff00');
        };

        this.ws.onmessage = (event) => {
            try {
                const messageData = JSON.parse(event.data);
                console.log('Received TikFinity Event:', messageData);
                this.handleTikFinityEvent(messageData);
            } catch (e) {
                console.error('Error parsing WebSocket message:', e);
                this.addEventToLog(`System Error: Could not parse message.`, '#ff0000');
            }
        };

        this.ws.onerror = (error) => {
            console.error('WebSocket Error:', error);
            this.addEventToLog('System Error: WebSocket connection error.', '#ff0000');
        };

        this.ws.onclose = () => {
            console.log('Disconnected from TikFinity WebSocket.');
            this.addEventToLog('System: Disconnected. Attempting to reconnect...', '#ffff00');
            // Optional: implement reconnection logic
            setTimeout(() => this.connectWebSocket(), 5000); // Reconnect after 5 seconds
        };
    }
