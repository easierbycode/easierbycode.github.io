Product Identification Challenge

  • Key problem: incoming samples often have no visible brand logos, no readable UPC, and only an internal code on the packaging — making identification extremely difficult without opening the item ‣
  • Clothes are a particularly hard category (e.g., can't tell if an item is a hoodie or joggers without opening the bag) ‣ ‣
  • Short-term workaround: check the account's order history, filter to "delivered" within the last 72 hours to narrow down likely matches — far fewer items to search than the full order history ‣
  • Once a listing is created and stored in the database, future matching of similar items becomes easier ‣
  • Live demo: searched "black" in order history, found Black Fabletics Men's Joggers as top candidate; physically inspected a mystery package and identified it as Hanes Ultimate Cotton hoodies (medium and large) by cross-referencing order dates ‣ ‣ ‣

Barcode Scanning Demo

  • Tested scanning a Hanes hoodie barcode through a custom PWA tool; the scan returned product name, size (medium), and category data ‣ ‣
  • Some products lacked standard barcodes (only QR codes or internal codes), limiting scan reliability ‣
  • eBay's scanning tool also tested — results were mixed; image data alongside barcode improves match accuracy ‣ ‣
  • Tool returns up to 10 candidate results; staff still need to confirm the correct match ‣ ‣

TikTok "Turn Into Commissions" Loophole

  • When viewing an order on the TikTok app, a "Turn your product purchases into commissions" prompt appears — without clicking the X to dismiss it, clicking into it redirects to the official product page with full listing data ‣ ‣ ‣
  • This gives access to the exact product title, price, specs, and images ‣
  • Dismissing the modal (clicking X) removes this option permanently for that account ‣
  • This is the current method being used to retrieve product metadata for listing creation ‣

Share Link → eBay Listing Tool (Key Feature Request)

  • Every TikTok affiliate product has a shareable link; the link redirects through a series of hops to a product page containing a product ID and full listing data ‣ ‣ ‣
  • Desired workflow: paste share link → system follows redirects → auto-fills an eBay listing with title, images, price, and specs → saves as inactive / quantity 0 ‣ ‣
  • When the physical item arrives, staff search by product name in eBay, locate the pre-built listing, and set quantity to 1 to activate ‣
  • Custom internal product ID (whose sample, shelf location) to be embedded in the listing ‣
  • Pre-creating listings from order history ("on its way" items) is also possible using this approach ‣

Intake & Warehouse Workflow

  • Proposed flow: intake → unboxing → processing/sorting → listing → physical inventory location → pick & ship ‣
  • Items will be sorted into owner-specific bins from the start (e.g., Brian's items go into Brian's bin) ‣
  • Chris developed 15–20 broad product categories to cover all incoming items ‣
  • Statuses planned for edge cases: items on hold, items not worth listing (set aside for review), items to donate ‣
  • Packaging considerations: resealable bags are preferred; shrink-wrapping may be worth investing in at scale ‣ ‣

Business Scaling Plans

  • Target: 5,000 units/week inbound, selling 1,000–2,000 units/week ‣
  • Carl is leading the warehouse operations program with the goal of dramatically scaling volume (described as 15–20x current autopilot intake) ‣ ‣
  • Performance-based pay model: people with TikTok accounts get paid per sample sent in; warehouse staff get paid per unit processed ‣ ‣
  • Estimated 15–20% of items will be unsellable (consumables, damaged, etc.) ‣
  • Unsellable items to be taken home by staff or donated (St. Vincent de Paul / Goodwill mentioned); potential to track donation dollar value for tax write-offs and PR ‣ ‣ ‣

Infrastructure & Server Stability

  • The system runs on a distributed multi-node setup with dynamic DNS failover — if one node goes offline, the other two vote on a new master and data is preserved ‣
  • Currently running on a Mac with 8GB RAM; any modest computer is sufficient ‣

Claude AI Integration (Idea Raised)

  • Suggestion to add a Claude skill so staff could use natural language to manage inventory hands-free (e.g., "Hey Claude, add these three things," "What's the net after the eBay listing?") ‣
  • Discussed keeping the MVP simple first before layering on AI features ‣ ‣
