TikTok Product Data System

  • A backend system is being built to aggregate TikTok Shop product data from multiple sources: creator side, agency side, and customer service order lists ‣ ‣
  • A fixture of ~355 orders was extracted from TikTok's customer service order list by paginating through all pages; this may represent a user's complete order history ‣ ‣
  • Product data is being pulled via a third-party scraping API ("scrape craters") at approximately $1.88 per 1,000 requests, or as low as $0.99/1,000 on bulk plans ‣ ‣
  • 33 products have been cross-referenced against TikTok so far with 100% accuracy on pricing ‣ ‣
  • Plan is to validate 100 products total before committing to a paid credit upgrade (~$47 for 25,000 credits) ‣ ‣
  • Prices will be pulled via third-party API rather than scraping TikTok directly to avoid rate-limiting or ban risk ‣
  • Product data fields available include: original price, real price, discount percentage, min/max SKU prices, sold count, review count, sample reviews, seller info, product category, images, and a direct TikTok shop URL ‣ ‣ ‣ ‣
  • Graylog is being used as the backend engine; described as highly powerful and accessible to non-technical team members for querying ‣
  • Cron-style event triggers and conditional logic ("if this, then that") can be set up in Graylog for automated workflows ‣
  • Competitor Caliata was identified as likely having internal TikTok data access via a former TikTok employee and their own AI-based ranking algorithm ‣ ‣

TikTok Affiliate Link Investigation

  • Affiliate links follow a short URL structure starting with `t/Z...` that appears to hash a product ID and creator user ID together ‣ ‣
  • If the slug formula can be reverse-engineered, affiliate links could be auto-generated for any creator+product combination without manual copy-pasting ‣
  • This would enable a Go High Level-style site where members could duplicate a full affiliate product catalog with their own links pre-built in ‣ ‣
  • A researcher has reportedly reversed much of TikTok's Android and iPhone API traffic including JWT token structure; if current, this could unlock the affiliate link formula and other hidden data ‣ ‣
  • Affiliate link generation is considered a future/V2 priority, not current scope ‣

Inventory Management System

  • A physical inventory system for product samples is being planned, separate from the TikTok data system but integrated with it ‣ ‣
  • Products will be tracked by bin and shelf location (e.g., A1, B2) using barcode/QR/UPC scanning via phone camera ‣ ‣
  • BLE 5.1 is being explored as a private, reliable communication link between a handheld scanner and a kiosk/workstation, with USB/Bluetooth fallbacks ‣ ‣
  • Product dispositions being considered: **available**, **checked out** (with owner tracking), **reserved**, **on hold** (e.g., creator still using), **discontinued**, **fire sale** ‣ ‣
  • Workflow: product arrives → scanned as "on hand but not inventoried" → goes through fulfillment intake → cleared as available or held ‣ ‣
  • Multi-user roles planned: admin, creators, and warehouse/stock workers ‣
  • Creator-specific inventory separation needed (e.g., Brian's samples vs. Chris's samples have different disposition rules) ‣
  • A POS-style UI was floated for fun (scan items like at a checkout), with a simpler internal-use version planned first; could potentially be productized and sold as a SaaS tool later ‣ ‣
  • Webhook integrations (eBay, Stripe, etc.) discussed for automated fulfillment notifications ‣ ‣
