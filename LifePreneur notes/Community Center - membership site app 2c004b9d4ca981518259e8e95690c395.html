Summary

Multi-Account Tracking Value Proposition

  • Primary motivation is the psychological impact of aggregated metrics across multiple accounts ‣ ‣
  • Example: 8 videos per account across 5 accounts = 40 total videos, which feels more productive than focusing on the per-account number ‣
  • No meaningful way to view combined analytics across multiple accounts in TikTok currently ‣
  • Users must manually log out and switch between accounts to view data ‣ ‣
  • This capability could be a unique differentiator, as no other communities or agencies are doing multi-account tracking and display ‣

Current Manual Tracking Methods

  • Team member manually screenshots individual account earnings daily and adds them up to calculate total commissions ‣ ‣
  • Average daily earnings range from $180-$1,300, averaging out to $16,000/month in commissions across accounts ‣
  • Experimented with tracking earnings by hour to identify patterns (e.g., slow mornings vs. strong afternoons) ‣ ‣
  • Theory exists about a "fill" phenomenon where accounts reach a certain daily amount regardless of timing ‣ ‣
  • Tracked correlation between video count and commissions: averaged 10 videos/day per account one month, 5.8 videos/day another month ‣
  • Gamification potential identified: setting minimums (e.g., 5 videos × 5 accounts = 25 videos minimum) ‣
  • Testing revealed no clear correlation between posting more videos and sales spikes ‣

Technical Approach: Scraping vs. API

  • API access through TikTok requires lengthy application process (1-2 months minimum) ‣
  • APIs require individual seller connections - can only pull data from sellers who grant permission ‣
  • No clear "agency-level" API that provides access to all data easily ‣
  • Decision made to focus on web scraping through browser extension instead of pursuing API initially ‣ ‣
  • Scraping can work on any page that can be accessed via URL, including mobile web versions ‣ ‣
  • Cannot scrape data from the native TikTok mobile app ‣ ‣ ‣ ‣

Data Availability and Limitations

  • Desktop/web analytics lack specific order timestamps that are visible in the mobile app ‣ ‣ ‣ ‣
  • Mobile app shows exact date and time of each order and which videos generated sales ‣ ‣
  • Need to investigate if timestamp data exists in source code even if not displayed in UI ‣ ‣
  • Export functionality exists for some data but doesn't always provide all needed information ‣ ‣
  • Scraper can capture data at regular intervals with timestamps, enabling intraday tracking over time ‣ ‣
  • Could provide historical comparisons like "are you ahead of yesterday at this hour?" though value is questionable given TikTok's unpredictability ‣ ‣

Shop-Level Analytics Need

  • Current analytics only show SKU-by-SKU data, not aggregated shop totals ‣
  • Example: Salute has ~30 SKUs (different flavors) but no way to see total Salute sales across all SKUs ‣ ‣
  • This limitation makes it difficult to track performance for brand-specific contests ‣
  • Shop data and product codes are available in different locations and can potentially be cross-referenced ‣ ‣ ‣

Contest and Campaign Tracking Requirements

  • Brands run contests with prizes (e.g., $1,000 bonus for $10,000 GMV) that allow account combination ‣
  • Currently very difficult to track performance toward contest goals without extensive manual work ‣
  • Users must rely on brand reporting, which is frequently inaccurate ‣
  • Vision: Create internal campaign manager where contests can be configured with specific parameters ‣ ‣ ‣ ‣ ‣
  • Key parameters needed:
    • Date ranges (e.g., calendar month) ‣
    • Specific SKU inclusion/exclusion ‣
    • Account combination rules ‣
    • Video date restrictions (e.g., only GMV from videos posted in contest month) ‣
  • System should automatically track all qualifying data across multiple accounts and provide exportable reports ‣ ‣
  • Contests are currently underutilized - mostly claimed by people already qualifying rather than driving new behavior ‣ ‣

Analytics Features and Metrics

  • Click-through rate (CTR) data is available and historically brands like Goalie looked for 4%+ CTR ‣ ‣
  • Could create valuable queries like "show all videos with at least one sale in last 30 days and their CTR" ‣
  • Tool should allow flexible data slicing: before noon vs. after, Tuesday vs. Wednesday, etc. ‣

TikTok PPS Score Impact

  • TikTok rolled out Product Performance Score (PPS) for sellers ‣
  • At score of 3.5 or below, sellers lose affiliate marketing eligibility ‣
  • Example: Goshi (Japanese exfoliating towel) dropped to 3.2, losing commission eligibility ‣
  • Creators' videos continue running as ads but earn no commissions ‣
  • Sellers face catch-22: need to run ads to improve score but can't pay commissions while score is low ‣
  • Score calculated based on negative reviews, return rate, and refund rate ‣
